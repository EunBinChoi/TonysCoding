클래스 만들 때
1. 프로그램에 따라 생성자 
여러개 제작 가능
>> 예를 들어
>> Car라는 클래스 제작
>> public Car() // 디폴트 생성자
>> public Car(int speed, int gear, String name) // new Car(100, 10, "adasd")
>> 본인이 매개변수로 어떤걸 받을 지에
따라서 데이터 타입과 갯수를 
맞춰서 제작할 수 있음

2. 특수한 경우를 제외하고 멤버변수 (필드)
는 private (캡슐화를 위해)으로 지정하고 이를 접근할 때는
public <클래스 이름> getter {
	return sadasd
},
public void setter(<클래스이름> s)
{
	this.s = s;
}
 함수를 만드는게 좋음!

3. 상속 >> extends
인터페이스 상속 >> implement
(*인터페이스는 클래스가 아님, 
기능적인 메소드들의 집합)

4. static (정적) >> main 함수보다 앞에 불림
클래스와 관련이 있음 (객체와 관련있지 않음)
static으로 설정된 필드나 메소드를 부를 때는
"클래스 이름.static필드" or "클래스 이름.static메소드"
(사실상 객체이름.필드 or 객체이름.메소드도 가능하지만 지양함)
(가능한 이유?? 객체에 static으로 지정된 필드/메소드의 주소를 알기 때문!)
ex) 생성된 객체의 갯수 (
private static int numberofCar = 0 >> 생성자에서 numberofCar ++)
public static int getNumberofCar() {
	return this.numberofCar;
}
